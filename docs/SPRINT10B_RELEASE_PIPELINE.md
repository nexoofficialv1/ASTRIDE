# Sprint 10B — Signed Android field-test and release pipeline

Sprint 10B makes Android builds reproducible on a clean GitHub runner.

## Delivered

- Strict validation of signing, Firebase, Maps, and runtime URL inputs.
- Runtime environment JSON generated from GitHub Variables; no production URL is hardcoded in source.
- Native Android scaffold bootstrapped separately for the selected app.
- Release keystore applied to generated Gradle Kotlin/Groovy projects.
- Semantic build name and monotonic Android build number support.
- Signed AAB plus split APK generation.
- Versioned filenames, SHA-256 file, and machine-readable release manifest.
- Optional GitHub Release publishing.

## Release gate

A production release fails before compilation when any required secret is absent, the keystore cannot decode, Maps/Firebase configuration is absent, or production endpoints are insecure/placeholders.

## Field-test order

1. Deploy API and confirm `/health` and `/ready`.
2. Configure Firebase Android apps for both package IDs generated by Flutter (`in.astride.astride_passenger` and `in.astride.astride_driver`).
3. Restrict the Android Maps key by package name and signing SHA-1/SHA-256.
4. Build Driver staging APK and test permissions/background tracking on physical devices.
5. Build Passenger staging APK and complete booking, OTP, push, tracking, and payment tests.
6. After sign-off, run production builds with a new build number.
